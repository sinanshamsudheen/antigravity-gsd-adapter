# Contributing to Antigravity GSD Adapter

Thank you for your interest in contributing! This adapter is a community project to help Antigravity users leverage the powerful get-shit-done workflow system.

## How to Contribute

### Reporting Issues

**For adapter-specific issues:**
- Installation problems
- Path reference errors
- Guide improvements
- Integration bugs

‚Üí Open an issue here with:
- Clear description of the problem
- Steps to reproduce
- Your environment (OS, Antigravity version)
- Relevant error messages

**For workflow/template issues:**
‚Üí Report to the [original get-shit-done repo](https://github.com/glittercowboy/get-shit-done/issues)

### Suggesting Improvements

Have ideas for better integration with Antigravity?

1. Check existing issues/discussions first
2. Open a new issue with the `enhancement` label
3. Describe:
   - What problem it solves
   - Proposed solution
   - Example usage

### Contributing Code

**Setup:**
```bash
# 1. Fork this repo
# 2. Clone your fork
git clone https://github.com/yourusername/antigravity-gsd-adapter
cd antigravity-gsd-adapter

# 3. Create a branch
git checkout -b feature/your-feature-name
```

**Making Changes:**
- Keep changes focused (one feature per PR)
- Test your changes thoroughly
- Update documentation if needed
- Follow existing code style

**Submit PR:**
1. Push to your fork
2. Create Pull Request
3. Describe what changed and why
4. Link related issues

### What We're Looking For

**High Priority:**
- Improved installation scripts
- Better error handling
- More examples
- Windows compatibility improvements
- Integration tips for specific project types

**Medium Priority:**
- Documentation improvements
- Video tutorials
- Community-contributed workflows
- IDE integration guides

**Low Priority:**
- Cosmetic changes
- Minor refactors without clear benefit

## Development Guidelines

### Code Style

**Shell scripts:**
- Use bash for scripts
- Include error handling (`set -e`)
- Add comments for non-obvious steps
- Test on Linux and macOS

**Documentation:**
- Use clear, concise language
- Include code examples
- Add screenshots where helpful
- Update table of contents if needed

### Testing

Before submitting:
1. Test installation script on fresh system
2. Verify all path references work
3. Check links in documentation
4. Run example workflows
